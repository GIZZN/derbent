# 🤖 Реализация AI анализа кандидатов в HR панели

## 📋 Обзор

Добавлена функциональность AI анализа кандидатов в HR панель согласно документу `HR_AI_ANALYSIS_GUIDE.md`. HR-специалисты теперь могут автоматически анализировать всех кандидатов на вакансию с помощью искусственного интеллекта.

## 🚀 Что было реализовано

### 1. API функции для AI анализа
- **Файл**: `src/lib/api/aiApi.ts`
- **Функции**:
  - `analyzeJobCandidates(jobId, hrToken?)` - анализ всех кандидатов на вакансию
  - `getJobAnalysisResults(jobId, hrToken?)` - получение результатов анализа
  - `checkHRAIServiceHealth(hrToken?)` - проверка доступности AI сервиса
- **Обновления**:
  - Поддержка Bearer token авторизации
  - Обработка специфичных HTTP ошибок (400, 403, 404, 500)
  - Fallback на cookies авторизацию

### 2. Хук для авторизации
- **Файл**: `src/lib/hooks/useAuthToken.ts`
- **Функции**:
  - `useAuthToken()` - получение общего токена
  - `useHRToken()` - получение HR токена
- **Источники токена**:
  - localStorage (приоритет)
  - cookies (fallback)
  - Redux store (планируется)

### 3. Обновленная HR панель
- **Файл**: `src/app/Components/UnifiedHRPanel.tsx`
- **Новые возможности**:
  - Кнопка "🤖 AI Анализ кандидатов" в заголовке панели
  - Отображение результатов анализа с рейтингами кандидатов
  - Состояния загрузки и обработка ошибок
  - Кнопка очистки результатов
  - Автоматическое получение HR токена

### 4. Стили для AI анализа

## 🎯 Как использовать

### В HR панели
1. Перейдите в HR панель (`/hr`)
2. Убедитесь, что вы на вкладке "Заявки кандидатов на вакансии"
3. Нажмите кнопку "🤖 AI Анализ кандидатов"
4. Дождитесь завершения анализа
5. Изучите результаты с рейтингами кандидатов

### Для тестирования
1. Перейдите на `/test-ai-analysis`
2. Проверьте статус HR токена в заголовке
3. Введите ID вакансии для анализа
4. Нажмите "🤖 Запустить анализ"
5. Изучите результаты

### Для тестирования Markdown
1. Перейдите на `/test-markdown`
2. Изучите интерактивный редактор
3. Попробуйте различные Markdown возможности
4. Посмотрите примеры AI анализа в Markdown

### Требования к авторизации
- **HR Token** должен быть сохранен в localStorage как 'hrToken' или 'authToken'
- **Bearer Token** автоматически добавляется в заголовки запросов
- **Cookies** используются как fallback если токен не найден
- **Права доступа** - HR может анализировать только свои вакансии

## 📊 Результаты анализа

AI анализ предоставляет:

### Для каждого кандидата:
- **Overall Score** (1-10) - общая оценка кандидата
- **Match Score** (0-100%) - соответствие требованиям вакансии
- **Fit Level** (low/medium/high) - уровень соответствия
- **Strengths** - сильные стороны кандидата
- **Weaknesses** - области для улучшения
- **Recommendations** - рекомендации для HR
- **AI Notes** - комментарии AI

### Общая статистика:
- Количество проанализированных кандидатов
- Время обработки
- Краткое резюме анализа (с поддержкой **Markdown**)

### Markdown поддержка:
- **AI заметки** отображаются с полной поддержкой Markdown
- **Сводка анализа** поддерживает форматирование
- **Заголовки, списки, таблицы** - все возможности Markdown
- **Код блоки** с подсветкой синтаксиса
- **Ссылки и изображения** в результатах AI

## 🔧 Технические детали

### API эндпоинты
- `POST /hr-ai/analyze-job-candidates` - запуск анализа
- `GET /hr-ai/job/:jobId/analysis` - получение результатов
- `GET /hr-ai/health` - проверка доступности

### Авторизация
- **Bearer Token** - основной способ авторизации
- **Cookies** - fallback для совместимости
- **Хук useHRToken()** - автоматическое получение токена
- **Поддержка localStorage** - сохранение токена

### Состояния компонента
- `isAnalyzing` - процесс анализа
- `aiAnalysis` - результаты анализа
- `analysisError` - ошибки анализа
- `selectedJobForAnalysis` - выбранная вакансия
- `hrToken` - токен авторизации HR

### Обработка ошибок
- **400** - Неверные параметры запроса
- **403** - Нет прав доступа к вакансии
- **404** - Вакансия не найдена
- **500** - Ошибка AI сервиса
- Проверка доступности сервиса
- Кнопка повторной попытки
- Информативные сообщения об ошибках

### Markdown рендеринг
- **react-markdown** - основная библиотека для парсинга
- **remark-gfm** - поддержка GitHub Flavored Markdown
- **Кастомные компоненты** - стилизация для AI результатов
- **Адаптивный дизайн** - корректное отображение на всех устройствах
- **Безопасность** - защита от XSS атак

## 🚨 Ограничения

- Максимум 100 кандидатов за один анализ
- Таймаут анализа: 30 секунд
- Лимит запросов: 10 анализов в час на HR
- Требуется авторизация HR
