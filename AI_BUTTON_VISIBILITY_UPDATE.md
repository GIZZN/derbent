# AI Button Visibility Update

## 🎯 Изменения

Кнопка AI помощника теперь показывается **только когда AI сервис активен**.

## 🔧 Что изменилось

### 1. Условное отображение кнопки
```typescript
{/* Floating AI Button - показывается только когда AI активен */}
{!isCheckingAI && aiServiceAvailable && (
  <button>...</button>
)}
```

### 2. Условное отображение модального окна
```typescript
{/* AI Modal - показывается только когда AI активен и модал открыт */}
{!isCheckingAI && aiServiceAvailable && isOpen && (
  <div className={styles.aiModal}>...</div>
)}
```

### 3. Состояние проверки AI
```typescript
const [isCheckingAI, setIsCheckingAI] = useState(true);
```

## 🚀 Логика работы

1. **При загрузке компонента** - `isCheckingAI = true` (кнопка скрыта)
2. **Проверка AI сервиса** - отправляется запрос для проверки доступности
3. **Если AI доступен** - `aiServiceAvailable = true`, `isCheckingAI = false` (кнопка показывается)
4. **Если AI недоступен** - `aiServiceAvailable = false`, `isCheckingAI = false` (кнопка остается скрытой)

## ✅ Результат

- ✅ Кнопка AI помощника показывается **только когда AI сервис активен**
- ✅ Нет мерцания или неожиданного появления кнопки
- ✅ Плавная проверка статуса AI сервиса
- ✅ Лучший UX - пользователь видит кнопку только когда она действительно работает

## 🎉 Готово!

Теперь AI помощник появляется только когда он действительно может помочь пользователю!
